Design Decisions & Implementation Notes
Search Bar
Height: 48px (min-h-[48px]) — prominent but not oversized
Background: bg-[var(--dusk)] — one step elevated from the night surface
Border: 1px solid var(--twilight) → var(--coral) on focus
Focus glow: ring-2 ring-coral/10 + shadow 0 0 20px coral/08
Border radius: rounded-xl (14px) — slightly more than card radius
Placeholder: font-mono text-sm text-muted "Search events, venues, series..."
Active text: font-mono text-sm text-cream
⌘K badge: hidden when focused, visible at rest (desktop only)
Dropdown
Surface: bg-dusk/97 — slightly lighter than night for elevation
Border: inherits from search bar, top excluded (seamless join)
Radius: 0 0 14px 14px — bottom corners match search bar
Max-height: 60vh — scrollable, keyboard shortcuts visible
Animation: animate-dropdown-in (existing keyframe)
Group stagger: animationDelay 0/40/80ms (existing pattern)
Selected row: bg-twilight + border-coral/25 + translateX(1px)
Result icon: 32px rounded-lg bg-twilight/70, category color
Filter Chips
Search chip: bg-coral/12 text-coral border-coral/30 — highest visual priority
Date chip: bg-gold/10 text-gold border-gold/25
Free chip: bg-neon-green/10 text-neon-green border-neon-green/25
Search icon inside chip: 11px coral svg, consistent with search bar
Font: font-mono text-xs (11px) — consistent with FilterChip component
Clear ✕: svg at 14px, same color as text at 60-70% opacity
Order: search chip always first (semantic priority)
Mobile Specifics
Full-screen overlay: fixed inset-0 z-50 bg-void — native modal feel
Header: back arrow (←) + search input + Cancel text
Input: no rounded-full — sharp nav bar feel, iOS-consistent
Results: direct fill (no dropdown), section headers with see-all
Touch targets: 44px minimum height on all rows
Trending: horizontal scroll, flex-nowrap, -webkit-overflow-scrolling
Pre-search on mobile: trending pills + category 2-col grid
Enter → same overlay collapses, browse state loads behind it
