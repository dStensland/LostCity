Section Module Interface

The contract every section module follows. Consistency at the interface level enables flexibility at the content level.

DetailSection Contract

// Every section module exports:
interface
id: SectionId // unique registry key
component: React.FC<SectionProps>
trait: (data: EntityData) → boolean // does the data support this section?
label: string // "Now Playing", "On View", "Lineup"
icon?: PhosphorIcon // section header icon
// The layout renders each section with:
interface SectionProps
data: EntityData // full entity payload (section picks what it needs)
portalSlug: string
accentColor: string // inherited from entity category
entityType: 'event' | 'place' | 'series' | 'festival' | 'org'
// Orchestrator defines a manifest:
type SectionManifest
= SectionId[] // ordered list — layout filters by trait at render

How the Pipeline Resolves

Manifest (cinema)
showtimes, about, connections, features, planYourVisit, nearby
→
Trait Filter
showtimes → ✓ has screenings
about → ✓ has description
connections → ✗ no festival link
features → ✓ has 3 features
planYourVisit → ✓ has admission
nearby → ✓ has coords
→
Rendered
1. ShowtimesSection
2. AboutSection
3. ConnectionsSection
3. FeaturesSection
4. PlanYourVisitSection
5. NearbySection
ConnectionsSection drops out because this cinema has no festival or org links right now.
During ATLFF, the trait fires and it appears at position 3 automatically.
