Data Fetching — Unified Hook

// Every orchestrator uses the same hook:
const { data, status, error } = useDetailData<EventData>({
entityType: 'event',
identifier: eventId,
portalSlug,
initialData, // from server render (SSR)
})
// Hook internals:
1. If initialData provided → skip fetch, use directly
2. Else → fetch from /api/{entityType}/{identifier}
3. Response envelope: { data, error, status }
4. Retry logic, cancellation, error states — all shared
5. Returns typed data for the specific entity type
// API routes stay the same — no backend changes needed:
/api/events/[id] → EventData
/api/places/by-slug/[slug] → PlaceData
/api/series/by-slug/[slug] → SeriesData
/api/festivals/by-slug/[slug] → FestivalData
/api/producers/by-slug/[slug] → OrgData
