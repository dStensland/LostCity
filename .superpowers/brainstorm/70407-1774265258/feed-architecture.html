Feed Architecture: Editorial Guide with Index Fallback

The feed curates ~25-40 events worth seeing. "See all 273" is always one tap away.

The key principle: the feed is a vertical scroll, not a horizontal carousel

The current Lineup uses horizontal scroll — every card is the same size, position doesn't communicate importance. The redesigned feed is vertical: scroll position = importance. Big things are big. Small things are small. You don't need to scroll sideways through 100 identical cards to find the good stuff.

Proposed Section Order

Each section earns its position by answering a question the user has at that point in their scroll.

What changed vs. current feed

1

Contextual Hero binds to the biggest event today

Cherry Blossom Festival IS the hero, not a tiny marquee strip. When no flagship event exists, falls back to atmospheric time-of-day hero. The hero responds to reality.

2

News module removed from default feed

Local news (crime, civic admin) moved to a dedicated "Today in Atlanta" page accessible from nav. The feed's job is to inspire going out, not inform about shootings.

3

Tonight's Picks is vertical + tiered, not horizontal carousel

Hero card (full-width) → featured row (3 cards) → standard rows (compact). Visual weight = importance. "See all 47" links to the full unfiltered index. Shows ~15-20 curated picks, not 273.

4

Noise filtered from the editorial feed (but accessible via "See all")

Rec center programming, YMCA classes, library tax help filtered from the curated picks. Still accessible via "See all 273" — never hidden, just not featured. The editorial feed shows ~25-40 events worth going to.

5

Planning Horizon quality-gated to festivals/tentpoles only

No more weekly open mics in "On the Horizon." Only events that are: tentpole, festival, multi-day, or importance=flagship. A section called "big events coming up" should only contain big events.

What stays the same

Regular Hangs (already great), Now Showing (clean), the visual design system, all existing components. This is a reorganization of information hierarchy, not a visual redesign.
