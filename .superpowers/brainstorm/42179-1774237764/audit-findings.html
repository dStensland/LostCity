Design System Audit: Where We Are vs. Where We Could Be

Pencil design spec vs. live code — 8 components audited, 10 cross-cutting issues found

The Three Layers of Drift

The gaps fall into three categories with different fix strategies:

1

Precision Drift (quick fixes, high impact)

Individual properties that don't match the spec. Wrong font weight, wrong color, wrong padding. Each is a 1-3 line fix.

Badge: font-medium → font-bold, +tracking, +uppercase
VenueCard: distance cyan → green
FilterChip: +bg-white/5 on inactive
FeaturedCard: 320px → 288px, 16px radius → 12px
DetailHero: title 20/30px → 24px
EventCard AM/PM: font-medium → font-bold
~20 line changes total across 8 files
2

Philosophy Drift (needs a design decision)

Structural choices where code diverged from the design intent. These need a call on which direction is right — the Pencil spec might be wrong too.

DetailStickyBar: floating pill w/ blur vs. flat bar (design says flat)
Body font vs mono: code over-applies monospace where design uses Outfit
FeaturedCard: completely different proportions (bloated vs. tight)
VenueCard: different image layouts (rail vs. thumbnail)
These are design decisions, not just fixes
3

Elevation Opportunities (beyond fixing drift)

The Pencil spec itself may not be ambitious enough. "Elite design" means going beyond matching what we drew — it means making the design system itself better.

Motion/animation system (hover, transitions, page enter)
Empty/error/loading states designed consistently
Micro-interactions (save animation, RSVP confirmation)
Dark mode refinement (current token values vs. ideal contrast)
Mobile-first gesture patterns (swipe, pull-to-refresh)
This is where "elite" lives — beyond consistency into craft

The question: Do you want to fix the drift first (layer 1+2), then elevate (layer 3)? Or skip straight to rethinking the design system at a higher level and fix drift as part of the rebuild?
